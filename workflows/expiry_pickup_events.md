# 到期 / 回收行事曆事件 SOP

> **目標**：每一筆「到期（租期結束）」與「回收」事件，都要能讓現場直接照著去收貨，
> 不用再回頭翻備忘錄。所以資訊必須完整且格式一致。

---

## 必備欄位（缺一不可）

每個 `【到期】` / `【上午/下午】回收` 事件的內文，**至少**要有：

| 欄位 | 說明 |
|------|------|
| **日期** | 事件本身的開始日期（= 租期到期日）。 |
| **姓名** | 客人姓名（含暱稱），對齊備忘錄。 |
| **品項** | **套組內容 ＋ 加購** 全部展開列出（見下方），這是最常漏的一欄。 |
| **地址** | 完整地址 ＋ 房號。 |
| **Google 地圖連結** | `📍 Google 地圖：https://www.google.com/maps/search/?api=1&query=<URL編碼地址>` |

其餘可保留：方案、租期區間、Email/電話。金額方向若有則寫 `收 / 給`（見 `telegram-daily-digest` 規則）。

### 標準內文格式

```
租期到期 葉庭綺（庭綺）
Email：vivi8601201@gmail.com
方案：C 套組 × 1年
品項：冰箱、洗衣機、微波爐、單人加大床架、床墊、4片窗簾組、可調光吸頂燈、電風扇、桌椅組、曬衣桿、電飯鍋
租期：2026/07/17 - 2027/07/16
地址：大阪市東成区大今里南6丁目15-21 1005
📍 Google 地圖：https://www.google.com/maps/search/?api=1&query=...
```

`品項：` 這行固定插在 `方案：` 那行的**下面一行**。

## 品項怎麼組（套組內容 ＋ 加購）

**套組基底**（定義同網站 `order.js` 的 `PLANS`）：

| 套組 | 內容 |
|------|------|
| **A 套組** | 冰箱、洗衣機、微波爐 |
| **B 套組** | A 套組 ＋ 單人床架、床墊（寬 100cm） |
| **C 套組** | A 套組 ＋ 單人加大床架、床墊（寬 120cm） |

**加購**：把該客人在備忘錄 / 配送事件 `加購：` 那行的東西全部接上去
（窗簾、吸頂燈、電風扇、桌椅組、電飯鍋、吸塵器、曬衣桿、和室桌…）。
加購品名對齊 `order.js` 的 `ADDONS`。

---

## ⚠️ 一定要看備忘錄的特例（不能照套組硬套）

備忘錄裡的客製/替換註記會改變實際品項，務必逐筆核對：

- **換掉基底家電**：例：冠廷「不要床架也不要冰箱」→ 品項就不能有床架、冰箱；
  涓「冰箱洗衣機微波爐換成桌椅組、吸塵器、室內掛衣架」→ 拿掉那三樣、加上替換品。
- **床型升級/降級**：例：陳昊恩 C 套組但「要單人床」→ 寫單人床（非加大）；
  Ling「升級雙人床」→ 寫雙人床。
- **指定顏色/機型**：例：Ling「白色微波爐」→ 品項直接寫「白色微波爐」。
- **後補/加送品**：例：Si-Yu 6/25 加送餐桌椅組 → 要補進品項。
- **贈送 / 未付**：例：音綺合計只收套組價但有附吸頂燈/窗簾/電風扇、王玉婷吸頂燈未付
  —— **實體有送到就要列入品項**（收貨時要收回），帳務狀態是另一回事。

## 操作方式

事件目前是**手動建立**的（webhook 只建「入住配送」事件，不建到期事件）。

- 月曆讀寫工具在 `lib/gcal.js`（`listEvents()` 可列事件；OAuth refresh-token，無 SDK）。
- 批次補 / 改內文：照 `lib/gcal.js` 的 auth 寫一次性 script，用 Calendar REST
  `PATCH /calendars/{id}/events/{eventId}`，body `{ "description": "..." }`。
  範例做法：依 `事件日期 + 姓名片段` 精準鎖定單一事件 → 確認舊內文 → 插入 `品項：` 行。
- **務必先 dry-run**（印出 before/after、確認每筆只 match 到 1 個事件）再 `--apply`。
- 具冪等性：若內文已有 `品項：` 就跳過，避免重複插入。
